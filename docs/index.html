How it worksCómo funciona

Three moving parts: workers on your machines, one coordinator, one API in front. Tres piezas: workers en tus máquinas, un coordinador y una API al frente.

01 · worker

Run a workerEjecuta un worker

Install the desktop app or the hydra-worker CLI. Point it at a local model (Ollama, llama.cpp, LM Studio, vLLM) or sign in to a provider you own (OpenAI, Anthropic, Gemini, and more). Keys go into a local vault — they are never serialized and never sent anywhere. Instala la app de escritorio o el CLI hydra-worker. Apúntalo a un modelo local (Ollama, llama.cpp, LM Studio, vLLM) o inicia sesión con un proveedor tuyo (OpenAI, Anthropic, Gemini y más). Las claves van a una bóveda local — nunca se serializan ni se envían a ningún lado.

02 · coordinator

It registers, admin approvesSe registra, el admin aprueba

The worker connects to the coordinator and advertises only its capabilities and models — never secrets. An admin decides which privacy levels each worker may handle; whatever a worker claims for itself is overridden, fail-safe. El worker se conecta al coordinador y anuncia solo sus capacidades y modelos — nunca secretos. Un admin decide qué niveles de privacidad puede atender cada worker; lo que un worker declare por sí mismo se sobreescribe, a prueba de fallos.

03 · api

Call one APILlama a una sola API

Clients speak the OpenAI API to the coordinator's front door. Each request is routed by capability, requested model, and privacy level to an eligible worker, which executes it and streams the result back. Los clientes hablan la API de OpenAI con la puerta de entrada del coordinador. Cada petición se enruta por capacidad, modelo solicitado y nivel de privacidad hacia un worker elegible, que la ejecuta y devuelve el resultado en streaming.

Privacy routingEnrutamiento por privacidad

Every job carries a privacy level. The coordinator enforces it when routing, and the worker re-checks it before running — defense in depth. Cada trabajo lleva un nivel de privacidad. El coordinador lo aplica al enrutar y el worker lo vuelve a verificar antes de ejecutar — defensa en profundidad.

privacyprivacidad eligible workersworkers elegibles
public local-model and external-provider workers workers con modelo local y con proveedor externo
private external only if the job permits it; otherwise local / org / internal externo solo si el trabajo lo permite; si no, local / org / interno
sensitive not external-provider by default (must have a local model) sin proveedor externo por defecto (debe tener un modelo local)
local_only must not use an external provider no puede usar un proveedor externo

One API, your infrastructureUna API, tu infraestructura

The front door speaks the OpenAI API — chat completions, streaming, tools/function calling and model listing — so existing clients and SDKs work by changing only the base URL. La puerta de entrada habla la API de OpenAI — chat completions, streaming, tools/function calling y listado de modelos — así que los clientes y SDKs existentes funcionan cambiando solo la URL base. 

# any OpenAI client works — just change the base URL
curl https://hydra.example.com/v1/chat/completions \
  -H "Authorization: Bearer $HYDRA_KEY" \
  -d '{
    "model": "llama3.1",
    "messages": [{"role": "user", "content": "hello"}],
    "stream": true
  }'
